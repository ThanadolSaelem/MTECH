# **1\. ภาพรวม Workflow ปัจจุบัน**

จากการประชุมกับทีมงานสามารถสรุป workflow ปัจจุบันที่มีปัญหาหลักได้ดังนี้

## **ขั้นตอนที่ 1 — รับลูกค้าและทำสัญญา**

* ลูกค้าติดต่อเข้ามาทาง LINE → แอดมินลงนัดคิวใน Google Sheet

* ลูกค้ามาหน้าร้าน → กรอกข้อมูลในโปรแกรมหลังบ้าน (โปรแกรมเช่าซื้อ) → พิมพ์สัญญา → ลูกค้าเซ็น

* ข้อมูลลูกค้าถูกพิมพ์ซ้ำอีกครั้งใน Google Sheet (Double Entry)

  **ปัญหาที่พบ:** แอดมินต้องนำข้อมูลจากโปรแกรมกลับมาพิมพ์ซ้ำใน Google Sheet อีกครั้ง เนื่องจากระบบหลังบ้านมีข้อจำกัด ดังนี้:

* **ขาดข้อมูลเพื่อคำนวณค่าคอมมิชชั่น:** ไม่มีช่องระบุชื่อแอดมินที่รับเคส

* **การระบุสาขาไม่ชัดเจน:** ระบบไม่มีช่องแยกสาขาโดยเฉพาะ (ต้องไปฝากไว้ในหมายเหตุ)

* **ไม่ระบุช่องทางการชำระเงิน:** ข้อมูลที่ดึงออกมาไม่บอกว่าลูกค้าจ่าย "เงินสด" หรือ "เงินโอน" ทำให้ตรวจสอบยอดเงินยาก

ขั้นตอนที่ 2 — เตรียมใบแจ้งหนี้

* ปริ้นท์สัญญาจาก Google Drive → ตรวจสอบข้อมูล (ชื่อ, ที่อยู่, เลขผู้เสียภาษี, ยอด, จำนวนงวด) 

* Copy ข้อมูลลง Excel Template ของ SME Move → Upload ขึ้นโปรแกรม (จำกัด \~50-60 ใบ/ครั้ง)

* แตกงวดรับชำระในโปรแกรมทีละใบ

           **การดึงข้อมูล:** Copy ข้อมูลจากข้อมูลระบบหลังบ้านที่ออกสัญญา  และมาใส่เชื่อมสูตรแล้ว ลงใน Excel Template ของโปรแกรม **SME Move**           

           **รายละเอียดที่ใช้ที่ต้องเชื่อมสูตรมา:** เลขที่ใบแจ้งหนี้ (INV), วันที่สัญญา, ข้อมูลลูกค้า (ชื่อ/ที่อยู่/เลขผู้เสียภาษี), และรายละเอียดสินค้า/เงื่อนไขการผ่อน  เพื่อให้  

            ข้อมูลมาครบถ้วนทั้งหมด

## **ขั้นตอนที่ 3 — ติดตามและรับชำระ**

* ดูรายการที่ครบกำหนดในแอปสัญญา → แจ้งลูกค้าผ่าน LINE     **การจัดการยอดค้างชำระโดยเจ้าหน้าที่** **ค้างชำระเกิน 5 วัน (Lock Mode)** / **ค้างชำระเกิน 15 วัน (Lost Mode & Legal Action)** **หน้าที่เจ้าหน้าที่:** ไล่ตรวจสอบรายชื่อสัญญาที่ค้างชำระในระบบหลังบ้าน ด้วยการไล่ดูทีละสัญญา  **การดำเนินการ:** เข้าระบบ **MDM** เพื่อทำการ **"ล็อกเครื่อง"** ของลูกค้ารายนั้นทีละราย 

1. ลูกค้าโอนเงิน → ส่งสลิปทาง LINE → ชะเอมบันทึกสลิปลง Google Drive         
   **การตรวจสอบเบื้องต้น:** \* เจ้าหน้าที่เข้าไปดูข้อมูลในสลิปและตรวจสอบในระบบหลังบ้านว่าเป็นการชำระ **"งวดที่เท่าไหร่"**   **กรณีระบบไม่อ่านสลิป:** เจ้าหน้าที่ต้องตรวจสอบยอดเงินจาก **Statement ธนาคาร** โดยตรง เพื่อยืนยันว่ามียอดเงินเข้าจริงก่อนดำเนินการขั้นต่อไป    **การเก็บหลักฐานในไฟล์ Google Sheets:** ลงข้อมูลในไฟล์ชื่อ **"SCB.\[เดือน.ปี\]"** (เช่น SCB.04.2026)**ข้อมูลที่ต้องลง:** เลขที่สัญญา (INV), งวดที่ชำระ/ปิดยอด/ค่าปรับ/ดาวน์, วันที่โอน, จำนวนเงินที่ได้รับจริง, และวันที่โอน 

* Download Bank Statement SCB → นำเข้า Google Sheet (เพิ่มชีตในไฟล์รับชำระ) เพื่อกรอกข้อมูลการรับชำระ   
* ดูข้อมูลในโปรแกรมเช่าซื้อ ว่ายอดรับโอนเงิน ในตาราง Bank Statement SCB แต่ละรายการ เป็นการชำระเงิน จากเลขที่ INVใด, เป็นเงินดาวน์/ค่างวดไหน/ปิดยอด, เงินโอน/เงินสด(กรณีฝากเงินสดเข้าธนาคาร), ค่าบริการ(เพิ่มเติม)เท่าไหร่(ถ้ามี), ค่าปรับเท่าไหร่และเป็นค่าปรับของงวดใด → พิมพ์ลงตาราง Bank Statement SCB Google Sheet ด้วยมือ

## **ขั้นตอนที่ 4 — ออกใบกำกับภาษี/ใบเสร็จเงินดาวน์,ค่างวด,ค่าบริการ(ถ้ามี) และใบเสร็จค่าปรับ**

* Match ข้อมูลการชำระเงิน ในตาราง Bank Statement SCB (\*1) กับ ตารางตัดรับชำระ (\*2)

\*1 นิยามของตาราง Bank Statement SCB คือตารางยอดเงินโอนเข้าบัญชีธนาคาร, ยอดรับเงินสด, ยอดเงินสดฝากธนาคาร โดยเพิ่มเติมคอลัมน์สำหรับกรอกข้อมูลอ้างอิงการรับชำระในแต่ละรายการ แยกตามเลขที่INV เป็นหลัก 

\*2 นิยามของตารางตัดรับชำระ คือตารางคุมยอดคงเหลือลูกหนี้ในแต่ละเลขที่สัญญา ยอดตัดรับชำระที่แสดงในตาราง เป็นการรวมยอดที่ชำระแล้วทั้งหมดในเดือนนั้นๆของแต่ละเลขที่สัญญา ใน1สัญญา อาจมีการชำระเงินเข้ามาหลายรายการในแต่ละเดือน และระบุวันที่ครบกำหนดค่างวดในเดือนนั้นๆ ส่วนรายละเอียดข้อมูลวันที่ที่โอนเงินเข้าบัญชี SCB หรือวันที่รับเงินสด/วันที่ฝากเงินสดเข้าบัญชีธนาคาร จะอยู่ในตาราง Bank Statement SCB ที่ชะเอมกรอกข้อมูลเพิ่มเติมทั้งหมด (เลขที่สัญญาเช่าซื้อ กับ เลขที่INV(ใบแจ้งหนี้) ใช้เลขที่เดียวกัน)

* เปิดใบกำกับภาษีใน SME Move ทีละใบ ตามวันครบกำหนด (\~38-40 ใบ/วัน)

* ตรวจ Statement  →  Match ข้อมูลการชำระเงินในตาราง Bank Statement SCB กับ ไฟล์ excel (Export file) ใบกำกับภาษีที่เปิดตามวันที่ครบกำหนดจากโปรแกรม smemove → ย้อนกลับมาแก้วันที่ในใบกำกับภาษีทีละใบ (ใช้เวลาในการแก้ไขแต่ละวันที่ที่มียอดรับชำระ \~ ครึ่งวัน)

* Logic วันที่: ลูกค้าจ่ายก่อนวันครบกำหนด → ใช้วันที่จ่ายจริง / จ่ายหลังหรือตรงวันครบ → ใช้วันที่ครบกำหนด

* เปิดใบกำกับภาษี/ใบเสร็จรับเงินค่าบริการเพิ่มเติมทีละใบ(ถ้ามี) เช่น ค่าบริการสัญญานอกสถานที่, ค่าบริการปลดล็อกเครื่อง โดยเปิดเรียงตามวันที่รับชำระค่าบริการ

* ค่าปรับ: ชะเอมแจ้งยอดค่าปรับผ่าน Excel → พี่นกเปิดใบเสร็จค่าปรับทีละใบ (\~100+ ใบ/เดือน, ไม่มี VAT) โดยเปิดเรียงตามวันที่รับชำระค่าปรับ

\*\* หมายเหตุ : การเปิดใบกำกับภาษีค่างวด ในโปรแกรม Peak วัตถุประสงค์คือเพื่อต้องการให้ การเปิดใบกำกับภาษี กับ การเปิดใบเสร็จรับเงิน สามารถเปิดเอกสารแยกจากกันได้ เพราะโปรแกรม smemove ไม่สามารถเปิดใบกำกับภาษีค่างวดและใบเสร็จแยกจากกันได้ค่ะ (เปิดใบกำกับภาษี/ใบเสร็จรับเงิน \= ตัดรับชำระหนี้แล้ว) ส่งผลทำให้กรณีลูกค้าจ่ายค่างวดหลังวันที่ครบกำหนด ยอดลูกหนี้คงเหลือในโปรแกรม smemove จะไม่ตรงกับความเป็นจริง เพราะค่างวดถูกตัดรับชำระหนี้ไปแล้ว ในวันที่เปิดใบกำกับภาษี/ใบเสร็จรับเงิน ตามวันที่ครบกำหนดค่ะ 

ระบบ ai ที่ใช้กับโปรแกรม Peak สำหรับเปิดใบกำกับภาษี/ใบเสร็จรับเงิน ที่ควรจะเป็นคือ 

1. เปิดใบกำกับภาษี/ใบเสร็จรับเงิน(เปิดรวมเป็นใบเดียวกัน) กรณีลูกค้าจ่ายค่างวดก่อนวันครบกำหนดเท่านั้น (ตัดรับชำระหนี้ในระบบแล้ว)

2. เปิดใบกำกับภาษี(ใบเดียว) กรณีลูกค้าจ่ายค่างวดหลังวันที่ครบกำหนด (ไม่ตัดรับชำระหนี้ในวันที่เปิดใบกำกับภาษี จะตัดรับชำระหนี้ในระบบ ก็ต่อเมื่อ เปิดใบเสร็จรับเงิน ตามวันที่ลูกค้าโอนจ่ายค่างวดหลังวันที่ครบกำหนดแล้วเท่านั้น

3. การรันเลขที่เอกสาร ให้รันเลขที่ใบกำกับภาษี และเลขที่ใบเสร็จรับเงิน แยกอิสระจากกัน (แม้ว่าจะเปิดเอกสารรวมในใบเดียวกันก็ตามค่ะ)

# **3\. สิ่งที่ไม่อยู่ใน Scope ครั้งนี้**

* OCR อ่านสลิปอัตโนมัติ — ซับซ้อน ใช้เวลาพัฒนา 1+ เดือน เหมาะเป็นโปรเจกต์ถัดไป

* เชื่อมต่อโปรแกรมหลังบ้าน (ระบบสัญญาเช่าซื้อ) โดยตรง

* ระบบจัดการสต็อก

* ระบบแจ้งเตือนลูกค้าทาง LINE

* การ Import ข้อมูลจากโปรแกรมสัญญา → Google Sheet (ลด Double Entry)

# **4\. ข้อมูลที่ต้องการก่อนเริ่มงาน**

* PEAK Account ที่รองรับ API (กรุณาเช็ก Plan กับ PEAK Support ก่อน)

* PEAK API Credentials: Client-Token และ User-Token

* Google Sheet ตารางรับชำระ (เซนเซอร์ข้อมูลสำคัญแล้ว) เพื่อ map field ให้ตรงกับ PEAK

* ตัวอย่างข้อมูล Bank Statement Excel (SCB) สัก 1 ไฟล์

* Confirm structure Google Sheet ชีตคืนเครื่อง ว่ามี field อะไรบ้าง

* Access Google Sheet (Editor) เพื่อทดสอบ

# **5\. หมายเหตุ**
